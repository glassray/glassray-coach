# Glassray Coach HTTP API

The full HTTP surface of the single Coach process (`http://127.0.0.1:<port>`, default
`5899`). Only the two ingest routes take auth — the local bearer key
(`$GLASSRAY_HOME/local-api-key`); every other route is unauthenticated **by design**:
the server binds `127.0.0.1` only, and every route enforces a loopback Host/Origin
guard (403 otherwise) as a DNS-rebinding defense — do not port-forward it to
untrusted networks. Long-running verbs reply `202 { runId, status }` and queue a
background run (a FIFO — one run executes at a time) — poll `GET /api/runs/:id`.
Back to the [README](../README.md).

## Ingest

| Route                             | Auth       | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| --------------------------------- | ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `POST /v1/traces`                 | Bearer key | OTLP/JSON envelope (`{ resourceSpans: [...] }`), 16 MiB cap, `application/json` — **`content-encoding: gzip`/`deflate`/`br` accepted** (the `@glassray/tracing` SDK gzips payloads ≥ 8 KiB). Spans are **merged into the stored trace by spanId** — a standard OTLP batch exporter that flushes one trace across several POSTs accumulates (incoming spans replace same-id stored spans, new ones append, and stored spans absent from the incoming batch are carried over), so a trace stays whole; the whole-trace-per-POST SDK path is unaffected (same spanIds ⇒ replacement). One malformed trace in a batch is skipped and logged (the batch's other traces still land); a wholly-malformed batch returns `400`; a datastore failure returns `503`. Replies `{}`. |
| `POST /api/public/otel/v1/traces` | Bearer key | Alias of the above (matches the Helix cloud ingest path).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `POST /v1/traces?label=<x>`       | Bearer key | Same ingest with a **run-label override**: every landed trace's `run_label` is set to `<x>` regardless of the envelope's own `environment` — how `pull --traces` tags real cloud traces `production`. Without the override, the label persists from the SDK `environment` (`glassray.environment`, `deployment.environment.name` alias; root span wins over resource).                                                                                                                                                                                                                                                                                                                                                                                                  |
| `GET /api/info`                   | —          | `{ name, version, ingestEndpoint, apiKey }`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |

## Traces

| Route                                              | Description                                                                                                                                                                                       |
| -------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `GET /api/traces?limit&offset&q&agent&status&flow&label` | `{ items, total }`, newest-first (offset paginated). `q` = case-insensitive substring on name/agent, `agent` = exact match, `status` = `error`\|`ok`, `flow` = only that flow's member traces, `label` = exact `run_label` match (one `glassray-coach run`'s corpus). |
| `GET /api/traces/:id`                              | `{ id, view }` — recomputed from the stored raw envelope on every read.                                                                                                                           |
| `GET /api/tail`                                    | SSE feed: `data: {"id":"<traceId>"}` per ingested trace, heartbeat every 25s.                                                                                                                     |
| `GET /api/stats`                                   | Totals (traces, tokens, errors, avg/p95 latency, est. cost + price-book `estCostIfMeteredUsd`) + a per-agent breakdown (both cost figures) + the known-agents list — powers the Overview KPIs and the CLI's `stats`.                                     |
| `GET /api/timeline`                                | `{ points: [{ t, traces, errors }], from, to }` — trace volume + errors bucketed into a fixed-width activity series (≥1-minute buckets) ending at the latest trace (the Overview activity chart). |

## Runs & queue

| Route                       | Description                                                                                                                                                                                                                                                                                                                                                                                        |
| --------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `GET /api/runs?limit`       | `{ items }` — recent runs of every kind (discovery / flows / eval / improver / classify / compare), newest first (default 20, max 200).                                                                                                                                                                                                                                                                      |
| `GET /api/runs/:id`         | `{ id, kind, status, error, stats, startedAt, finishedAt }`. Status lifecycle: `queued → running → done`\|`error`. While a **discovery or eval** run is `running`, its `stats` carries live progress `{ scanned, total }` (other kinds don't publish progress; terminal stats replace it on finish, and an eval run's terminal stats record `judgeModel`).                                         |
| `POST /api/runs/:id/cancel` | Cancels a **queued or active** run — a queued run is dropped from the FIFO, an active run's provider call is aborted (`409` if the run is neither).                                                                                                                                                                                                                                                |

## Flows

A flow's deterministic `selector` matches on agent / name / intent / status / pinned
trace ids, AND-combined; its plain-language `rule` is classified by the debounced
background sweep on the light tier — idempotent via a `classified_at` watermark and
budget-guarded.

| Route                      | Description                                                                                                                                                                                                                |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `POST /api/flows/run`      | **Discover flows from code** → `202 { runId, status }`. Body `{ codeRoot? }`: a read-only tool-using agent (Read/Grep/Glob, rooted at `codeRoot`) reads the agent's source and maps its flows + their rules; each rule lands as a `source: "code"` eval with file anchors, and traces attach afterward via the classify sweep. The CLI resolves `codeRoot` from `glassray.yaml`; the web button sends none, so the server falls back to the `glassray.yaml` in its launch cwd — `400` when neither yields one (except on `mock`). |
| `GET /api/flows?status`    | `{ items, unclassified }` — flows with parsed selectors + live member counts (`status` = `active` (default)\|`archived`\|`all`); `unclassified` = traces still awaiting the classify sweep.                                |
| `POST /api/flows`          | `{ name, description?, selector?, rule?, classify?, createdBy? }` → `201 { id, memberCount, llmBackfill }`. Needs a selector, a rule, or both; `classify: 'llm'` needs a rule. A new rule-defined flow re-opens the newest ~100 traces for the sweep (`llmBackfill`). |
| `GET /api/flows/:id`       | The definition + its newest members (≤ 100, with `assignedBy`/`confidence`) + the assertion rules scoped to it (with `source`, `anchors`, `threshold`).                                                                    |
| `PATCH /api/flows/:id`     | Partial update (`name`/`description`/`selector`/`rule`/`classify`/`status: active\|archived`) → the updated detail. A changed selector re-materializes memberships; a new/changed llm rule triggers the bounded backfill.  |
| `DELETE /api/flows/:id`    | Hard delete — memberships go with it; attached evals detach (become global).                                                                                                                                               |
| `GET /api/flows/:id/audit` | Classification-quality view: a newest-members sample, the low-confidence LLM assignments, and `counts { members, lowConfidence, unclassifiedStoreWide }`.                                                                  |

## Deviations & fixes

| Route                                          | Description                                                                                                                                                                                                                                       |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `POST /api/discovery/run`                      | `{ sampleSize?, flowId? }` → `202 { runId, status: queued\|running }`. Runs **queue** — never `409`-busy; a duplicate request returns the run already queued/active for the same scope. A `flowId` scopes the sample to that flow's members (`404` if unknown). |
| `GET /api/deviations` · `/api/deviations/:id`  | Recurring deviation types + their per-trace examples (label, severity, evidence, trace link). Each carries a `status` (`open`\|`resolved`); the list adds `hasFix`, the detail adds the generated `fixMarkdown` (+ `fixModel`, `fixGeneratedAt`). |
| `POST /api/deviations/:id/fix`                 | Generate a fix for the deviation → `202 { runId, status }` (queued; `404` if the deviation is unknown). The finished run stores `fixMarkdown` on the deviation row.                                                                               |
| `POST /api/deviations/:id/resolve` · `/reopen` | Flip the deviation's `status` between `resolved` and `open`.                                                                                                                                                                                      |

## Evals (assertion rules)

An eval is an **assertion rule**: a judged predicate (`text`) over a trace. Every rule
is active — it autoruns once its flow accrues `autorunThreshold` new members, and gates
`glassray-coach check` at its `threshold` pass rate. A rule carries its provenance
(`source`): `code` (read from the repo, with `anchors: [{ file, symbol?, line? }]`) or
`promoted` (hand-written, or saved from a deviation). There is no in-app lifecycle —
"approval" is git review of `glassray.yaml`, and removal is deletion.

| Route                     | Description                                                                                                                                                                                                                                                                                                                                    |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `GET /api/evals` · `/api/evals/:id` | Saved rules (with `name`, `text`, `source`, `anchors`, `threshold`, `judgeModel`, artifact `slug`) + their latest pass/fail rollup + regression count; the detail adds the per-trace verdicts (regressions first) and the run history.                                                                                               |
| `POST /api/evals`         | `{ deviationId, flowId? }` (save a deviation as a `promoted` rule — idempotent) or `{ name, text, description?, flowId?, anchors?, autorunThreshold?, threshold?, judgeModel? }` (hand-written; supplying `anchors` makes it `source: "code"`) → `201 { id }`. A `flowId` scopes runs to that flow's members.                                  |
| `PATCH /api/evals/:id`    | `{ flowId?, anchors?, autorunThreshold?, threshold?, judgeModel? }` → the updated detail (`flowId: null` detaches the eval; setting `anchors` re-derives `source`; the rule text itself is immutable via PATCH — the artifact import can rewrite it).                                                                                          |
| `POST /api/evals/:id/run` | `{ sampleSize?, model?, traceIds? }` → `202 { runId, status }` (queued). `traceIds` **pins the corpus** (the deterministic `check --fixtures` path); otherwise sample size: explicit override > the flow selector's `limit` > 20. `model` overrides the judge for this run (falls back to the rule's `judgeModel`, then the light default; recorded in the run's `stats.judgeModel`). |
| `DELETE /api/evals/:id`   | Remove an eval and its stored verdicts.                                                                                                                                                                                                                                                                                                        |

## The rule artifact & compare

`glassray.yaml` is the flows + rules serialized as a
versioned file; identity is the `(kind, id-slug)` pair, never a row id. The CLI verbs
`pull` / `push` / `check` / `compare` are loopback callers of these routes.

| Route                        | Description                                                                                                                                                                                                                                                                             |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `GET /api/export`            | `{ artifact, yaml }` — active flows + their rules in the artifact shape. Stamps derived slugs back onto rows that had none (durable identity).                                                                                                                                          |
| `POST /api/export`           | `{ baseYaml?, artifact? }` → `{ artifact, yaml }`. Export with **local-only preservation**: the base file's per-flow `run` recipes + `project`/`fixtures` refs are overlaid onto the fresh portable sections (what `glassray-coach pull` uses so the harness-authored recipe survives). `artifact` substitutes an external portable source — the cloud-pull path. |
| `POST /api/artifact/parse`   | `{ yaml }` → `{ artifact }` — validate + structure a glassray.yaml document (the zero-dependency CLI reads run recipes through this).                                                                                                                                                   |
| `POST /api/import`           | `{ yaml? \| artifact?, apply?: bool, prune?: bool }` → the reconcile plan (`actions: [{ op: create\|update\|prune\|noop, kind, id, changes }]`, `summary`). `apply: false` (default) is the dry-run; under `prune: true` a pruned **flow is archived** and a pruned **rule is deleted** (rules have no archived state). A legacy `state:` key in the file fails loudly with a migration hint — the proposed/watched lifecycle is retired. Idempotent — re-applying the same file plans all-noop. |
| `GET /api/flows/:id/fixtures?limit` | `{ flow: { id, name, slug }, items: [{ traceId, raw }] }` — the flow's newest members' stored OTLP envelopes (default 20, max 200), for `pull --as-fixtures`. Re-POSTing an envelope to `/v1/traces` reproduces the trace (idempotent by trace id).                              |
| `POST /api/compare`          | `{ baseline, candidate, flowId?, sampleSize? }` where each corpus is `{ traceIds }` \| `{ label }` \| `{ agent }` \| `{ flowId }` → `202 { runId, status }`. Runs the suite over both corpora — the flow's rules when `flowId` is given, else only the **global** (flow-detached) rules, never flow-scoped ones; the finished run's `stats` carries the report: per-rule pass rates + delta + `regressed`, and each side's traces/tokens/`estCostUsd` (provider-blended real spend) + **`estCostIfMeteredUsd`** (price-book by each trace's primary model — non-zero even on free providers) + avg-latency, plus `costIfMeteredDeltaUsd`. Verdicts are never written to `eval_results`. |

## Experiments

An experiment is the **durable container for one question** ("can we switch to
Haiku?"): it wraps a baseline-vs-candidate compare and keeps the generated report —
data only, no go/no-go call. Experiments are records; they are never part of
`glassray.yaml`.

| Route                               | Description                                                                                                                                                                                                                                                                        |
| ----------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `POST /api/experiments`             | `{ question, flowId? }` → `201 { id }` — open an experiment (`status: open`, no report yet). `404` if the flow is unknown.                                                                                                                                                         |
| `GET /api/experiments?flowId`       | `{ items, total }` — stored experiments, optionally scoped to a flow.                                                                                                                                                                                                              |
| `GET /api/experiments/:id`          | The record: `{ id, flowId, question, status: open\|running\|concluded, baselineLabel, candidateLabel, runId, report, createdAt, concludedAt }`.                                                                                                                                    |
| `POST /api/experiments/:id/report`  | `{ baseline?, candidate? }` (run labels; both default to the two newest labels, candidate newest) → `202 { runId, status, experimentId, baseline, candidate }`. Queues the compare over the experiment's flow's rules (the global rules when the experiment is unscoped) and stores the report on conclusion: a prose `summary`, `regressions`, `costDeltaUsd`, per-regressed-rule `failing` detail, and the embedded compare report. `400` when no two labels exist yet. |

## Settings, LLM & usage

| Route                                       | Description                                                                                                                                                                           |
| ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `GET /api/llm`                              | `{ provider, ready, reason }` — what discovery/flows will use.                                                                                                                        |
| `POST /api/replay`                          | `{ prompt, system?, model?, temperature? }` → `{ output, provider, model }` — re-issue an edited LLM call as free text (`402` if the spend cap is reached, `502` if the provider is unreachable). |
| `GET /api/usage`                            | Coach's own LLM spend vs the budget, broken down by model + kind. Each model row (and the `spentIfMeteredUsd` total) also carries the **price-book** "cost if metered" estimate, so a free-provider run still answers "is it cheaper?".            |
| `POST /api/usage/reset`                     | Clear the usage ledger.                                                                                                                                                               |
| `GET /api/settings` · `PATCH /api/settings` | Read / update the persisted dashboard settings (LLM provider, heavy/light model ids, budget). Saved to `$GLASSRAY_HOME/settings.json` (`chmod 0600`); API keys are never stored here. |
